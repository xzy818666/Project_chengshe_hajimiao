# 赛博功德银行 — 修改报告

> 生成时间：2026-06-10
> 涵盖范围：教程系统重构、建筑热区精确化、HUD 布局修复、高亮定位优化

---

## 一、修改概述

本次修改围绕 **「新手引导体验」** 和 **「主界面交互精确度」** 两大主题展开，共涉及 8 项核心改动，覆盖 6 个源文件。核心目标：

1. 将原来的弹窗式教程（`TutorialDialog`）替换为 **内嵌遮罩式教程**（`TutorialManager` + `TutorialOverlay`），支持实时高亮游戏界面元素并引导操作。
2. 将功德堂四角建筑的点击热区从 **覆盖整个建筑** 精确缩小为 **只覆盖建筑门/入口位置**，避免误触。
3. 修复教程气泡在对话框模式下被截断、高亮框偏移等显示问题。
4. 将修炼等级增益信息整合进右侧 HUD，消除与法器图标的重叠。

## 二、详细改动清单

### 2.1 TutorialManager + TutorialOverlay 内嵌教程系统（核心重构）

**状态**：✅ 已完成，38 步

**改动动机**：
- 原 `TutorialDialog` 为独立弹窗（W6），通过 `QStackedWidget` 分页展示，无法与真实游戏界面交互。玩家看完后进入游戏仍需自己摸索，教程与实战脱节。
- 新系统采用 **遮罩层 + 高亮框 + 气泡对话框** 的实时引导模式，玩家在实际游戏界面中按步骤操作，真正做到「边玩边学」。

**架构设计**：

| 类 | 类型 | 职责 |
|---|---|---|
| `TutorialManager` | 单例 `QObject` | 维护 38 步教程状态机，处理步骤切换、等待条件、信号通知 |
| `TutorialOverlay` | `QWidget` | 全屏遮罩层，绘制半透明暗色背景、高亮镂空区域、气泡对话框 |
| `TutorialStep` | 结构体 | 每步配置：文本、目标控件名、等待类型、模式（主界面/对话框） |

**关键行为**：
- **步骤模式**：
  - `next`：点击「继续」按钮进入下一步
  - `click_area`：等待用户点击指定区域（如建筑按钮、云海、标签页）
  - `click_cloud`：专用于云海点击，允许鼠标穿透
- **高亮机制**：所有带 `targetWidget` 的步骤都会显示金色高亮框，但 **仅 `click_area` / `click_cloud` 步骤会禁用目标控件的鼠标事件**（通过 `WA_TransparentForMouseEvents` 控制），普通步骤只高亮不拦截操作。
- **对话框支持**：教程可跟随进入 `BankDialog`、`ExchangeDialog` 等子对话框，气泡自动调整位置避免溢出屏幕。

### 2.2 气泡截断修复（`TutorialOverlay::positionBubble()`）

**问题**：对话框模式下气泡文本较长时，气泡宽度超出对话框边界，导致文本被截断或完全不可见。

**修复方案**（三重保障）：
1. **强制布局刷新**：气泡内容更新后调用 `layout()->activate()` + `adjustSize()`，确保尺寸计算基于最新文本。
2. **字体降级**：若文本总长度超过阈值，自动将字体从 14px 降至 12px，减少宽度需求。
3. **阴影 padding 补偿**：计算气泡位置时增加 `+20px` 水平 padding，避免阴影效果导致文本贴边。

**相关文件**：`tutorialoverlay.cpp`

---

### 2.3 鼠标事件穿透修复（`WA_TransparentForMouseEvents`）

**问题**：教程期间高亮框覆盖的控件无法点击，导致 `click_area` 步骤结束后，玩家仍无法与目标控件交互。

**修复方案**：
- `WA_TransparentForMouseEvents` 仅在 `click_area` 或 `click_cloud` 类型的步骤中启用，使鼠标事件穿透遮罩层到达目标控件。
- 普通 `next` 步骤或纯信息展示步骤不启用此属性，遮罩层正常拦截点击。
- `click_area` 步骤结束后自动恢复控件正常交互。

**相关文件**：`tutorialoverlay.cpp`

### 2.4 修炼等级 HUD 整合（`MeritHall::updateHUD()`）

**问题**：修炼等级显示原先位于 HUD 左下角，与底部莲花台法器图标区域重叠，视觉混乱。

**修复方案**：
- 将修炼等级显示移至 **右侧 HUD 面板**（`rightHudLayout`）内部，与下世储备、共鸣信息同一区域。
- 显示内容扩展：除等级称号和进度条外，新增 **等级增益明细**：
  - 效率加成（%）
  - 暴击率加成（%）
  - 自动收益加成（%）
  - 最大杠杆加成（×）
- 使用 `QProgressBar` 显示经验进度，数值标签紧跟其后。

**相关文件**：`merithall.cpp`

### 2.5 教程高亮目标定位优化（`TutorialOverlay::findTutorialTarget()`）

**问题**：`findTutorialTarget()` 遍历同名控件时，常命中 `.ui` 文件中默认尺寸很小（如 10×10）的隐藏按钮或占位控件，导致高亮框偏移严重或几乎不可见。

**修复方案**：
- 增加 **最小尺寸过滤**：只选择可见（`isVisible()`）且宽高均大于阈值（主界面 50×50，对话框 30×30）的控件。
- 遍历逻辑：优先查找 `parentWidget()->findChildren<QWidget*>(name)`，匹配第一个满足尺寸要求的控件。
- `resizeEvent()` 中实时重新查找当前步骤目标，确保窗口缩放后高亮框位置正确。

**相关文件**：`tutorialoverlay.cpp`

### 2.6 BankDialog 交互式教程（`click_area` 步骤）

**问题**：原教程中「尝试借一笔钱」步骤为纯文本说明，玩家看完仍需自行操作，缺乏引导感。

**修复方案**：
- 将善财司教程拆分为两步交互式引导：
  1. **步骤 11**：`targetWidget="bankLoanTab"`, `waitFor="click_area"` → 提示玩家点击「功德借贷」标签页。
  2. **步骤 12**：`targetWidget="bankBorrowBtn"`, `waitFor="click_area"` → 提示玩家点击「借贷」按钮。
- 在 `BankDialog` 构造函数中连接信号：
  - `tabWidget::currentChanged` → 切换到「功德借贷」页时触发 `notifyAreaClicked("bankLoanTab")`
  - `borrowBtn::clicked` → 点击时触发 `notifyAreaClicked("bankBorrowBtn")`
- 高亮框精确覆盖标签页和按钮区域。

**相关文件**：`bankdialog.cpp`

### 2.7 建筑按钮热区精确化（`MeritHall::updatePavilionPositions()`）

**问题**：四角建筑按钮（幻缘所、善财司、法器阁、成就）的热区覆盖整个建筑区域，玩家容易在非门区域误触打开界面。

**修复方案**：
- 热区从「全建筑覆盖」改为「只覆盖建筑门/入口位置」，基于 `door_glow.png` 的中心坐标计算。
- 热区尺寸比 glow 素材略大（方便点击）：

| 建筑 | 门中心坐标（2560×1440） | 热区尺寸 | 备注 |
|---|---|---|---|
| 幻缘所（左上） | (821, 560) | 220×300 | 小门，热区适当放大 |
| 善财司（右上） | (1777, 560) | 220×300 | 小门，热区适当放大 |
| 法器阁（左下） | (709, 930) | 280×350 | 大门，热区较宽 |
| 成就（右下） | (1872, 930) | 280×350 | 大门，热区较宽 |

- 热区按窗口宽度与 2560 的缩放比例实时计算，窗口缩放时自动调整。
- 旧的全建筑热区代码（`w * 0.66, h * 0.10` 等）已删除。

**相关文件**：`merithall.cpp`

## 四、关键实现细节备忘

### 4.1 教程步骤数据结构

```cpp
struct TutorialStep {
    QString text;           // 气泡文本（支持 HTML）
    QString targetWidget;   // 目标控件 objectName
    QString waitFor;        // "next" / "click_area" / "click_cloud"
    bool isDialogMode;      // true = 在子对话框中显示
    QString dialogName;     // 对话框标识（如 "BankDialog"）
};
```

### 4.2 高亮框绘制逻辑

- 在 `TutorialOverlay::paintEvent()` 中绘制两层：
  1. 全屏半透明暗色遮罩（`rgba(0, 0, 0, 0.6)`）
  2. 使用 `QPainter::setCompositionMode(QPainter::CompositionMode_Clear)` 在目标控件位置「镂空」一个矩形，露出下方高亮内容
- 高亮框边框：2px 金色实线 `#FFD700`
- 高亮框内发光：`QGraphicsDropShadowEffect` 或 `QPainter::drawRect` 多次描边模拟

### 4.3 气泡定位策略

| 场景 | 策略 |
|---|---|
| 目标在屏幕上半部分 | 气泡放在目标下方 |
| 目标在屏幕下半部分 | 气泡放在目标上方 |
| 目标在屏幕左半部分 | 气泡左对齐目标左边缘 |
| 目标在屏幕右半部分 | 气泡右对齐目标右边缘 |
| 对话框模式 | 气泡强制放在对话框左下角，避免溢出父窗口 |

### 4.4 建筑热区缩放公式

```cpp
double scale = static_cast<double>(w) / 2560.0;
int pw = static_cast<int>(220 * scale);  // 热区宽度
int ph = static_cast<int>(300 * scale);  // 热区高度
int px = static_cast<int>(821 * scale) - pw / 2;  // 左上角 x（门中心对齐）
int py = static_cast<int>(560 * scale) - ph / 2;  // 左上角 y（门中心对齐）
```
